
\chapter{Metodología}

Las circunstacias profesionales del alumno en el periodo que se realizó el
proyecto hicieron que se tomaran de forma natural muchos de los métodos de
las metodologías ágiles, de manera que permitiera un óptimo y flexible reparto 
de tiempos entre todas las tareas a desarrollar a lo largo del proyecto.

Así se terminó utilizando de manera natural muchas de las normas de este tipo
de métodos:

\begin{itemize}
  \item Simplicidad: desde el primer momentos se buscó hacer el proyecto de la 
	manera más sencilla posible.
  \item Desarrollo iterativo e incremental: una vez se fue teniendo las 
	funcionalidades principales, construyendo alrededor de los casos de uso
	principales pequeñas utilidades para enriquecer la aplicación.
  \item Propiedad del código compartida: desde las primeras lineas, todo el 
	código ha estado publicado en el 
	Subversion\footnote{\url{http://subversion.tigris.org/}}  del proyecto, 
	al alcance en cualquier momento de las tres personas involucradas en el 
	proyecto.
  \item Refactorización del código: de manera frecuente partes del código del 
	proyecto han sido sometidas a técnicas de refactorización para mejorar 
	partes que no había sido codificadas de la mejor de las maneras.
  \item Pruebas unitarias (FIXME: mentira, pero es una tarea que tengo que abordar 
	con PyUnit)
  \item Frecuente interacción del equipo de programación con el cliente o usuario:
	en este caso la figura del cliente ha tenido la forma del co-director del
	proyecto (Diego Berrueta), con el que se han mantenido frecuentes encuentros
	para tratar aspectos concretos del proyecto.
\end{itemize}
