Motivation and Objectives

  • Effective large-scale mining of the social web requires the availability of big amounts of well-defined data. The semantic web provides a convenient platform to publish and consume these data.

  • Objectives:

    • Produce large amount of machine-processable data in RDF
    • Care about the quality of the data, linking it to other sources
    • Reuse existing ontologies, such as SIOC and FOAF

Social Online Communities with Semantics

SIOC (Semantically Interlinked Online Communities) provides the main concepts and properties required to describe information from online communities (blogs, forums, mailing lists, wikis, etc.) on the Semantic Web.

Techniques for Mining Online Communities

We consider two approaches to populate the SIOC ontologies with instances:

  • Intrusive techniques
    • Require access to the back-end data in RAW format
    • case study: SWAML
  • Unintrusive techniques
    • Mine the Web representation of the community
    • case study: HTML scraping with XSLT

SWAML, mailing lists meet the Semantic Web

SWAML is a Python script that parses a mailbox, uses Web services (such as SWSE or Sindice) to locate FOAF profiles to enrich the dataset, and outputs the information in RDF.

HTML Scraping

  • a simple approach for HTML scraping:
    • step #1: crawl the community Web site
    • step #2: tidy the markup
    • step #3: transform XHTML pages with XSLT
  • different outputs:
    • RDF/XML
    • XHTML+RDFa

Experiment

The two techniques have been tested on Free Software communities:

Collected around of 30 million RDF triples from mailing lists

Linked to more than 4,000 FOAF profiles extracted from Advogato

Common issues

  • Same person, multiple identities
  • Hashed e-mail addresses
  • Repeated primary keys
  • Flat threads
  • Pagination

Conclusions and future work

  • Conclusions:
    • Existing ontologies, such as SIOC and FOAF, can describe online communities
    • Each scenario dictates diferent requisites and thus a different technique
  • Future work:
    • Incorporate new transformation technologies, such as XSPARQL
    • Improve smushing (duplicated instances detection)
    • Balance information integration with privacy issues
    • Study complementary techonologies, such as Semantic Email
